# Terra Consulting LLC - Environment Variables

## Backend (.env)

Create a `.env` file in the `server` directory with the following variables:

```bash
# Database
DATABASE_URL="file:./dev.db"

# JWT Secret (generate a secure random string)
JWT_SECRET="your-secure-random-secret-here"

# Allowed Origins (comma-separated)
ALLOWED_ORIGINS="http://localhost:3000,https://your-vercel-app.vercel.app"

# Environment
NODE_ENV="development"

# Server Port
PORT=5000
```

## Frontend (.env.local)

Create a `.env.local` file in the `client` directory with the following variables:

```bash
# API URL
NEXT_PUBLIC_API_URL="http://localhost:5000"

# Mapbox Token
NEXT_PUBLIC_MAPBOX_TOKEN="your-mapbox-token"

# WhatsApp Number (Kenya format)
NEXT_PUBLIC_WHATSAPP_NUMBER="254700000000"
```

## Production Environment Variables

### Render (Backend)
Set these in your Render dashboard after deployment:
- `NODE_ENV`: `production`
- `DATABASE_URL`: `file:/var/data/dev.db`
- `JWT_SECRET`: (auto-generated by Render)
- `ALLOWED_ORIGINS`: Your Vercel frontend URL (e.g., `https://terra-consulting.vercel.app`)
- `PORT`: `5000`

### Vercel (Frontend)
Set these in your Vercel project settings:
- `NEXT_PUBLIC_API_URL`: Your Render backend URL (e.g., `https://terra-consulting-api.onrender.com`)
- `NEXT_PUBLIC_MAPBOX_TOKEN`: Your Mapbox token
- `NEXT_PUBLIC_WHATSAPP_NUMBER`: `254700000000`
